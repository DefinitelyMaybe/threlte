The `<Canvas>` component is the root of your Threlte scene. It provides
contexts that all other components and many hooks depend on. This means they
need to be **child components** to the `<Canvas>` component.

<Tip
  type="tip"
  title="Structuring Your App"
>
  Check out our [guide on structuring your app](/docs/learn/basics/app-structure) for a fail-safe
  app architecture recipe.
</Tip>

## Size

By default, the `<Canvas>` takes up `width:100%` and `height:100%` as well as
resizes if changes occur.

When resizing, [offset sizes](https://developer.mozilla.org/en-US/docs/Web/API/CSS_Object_Model/Determining_the_dimensions_of_elements#how_much_room_does_it_use_up)
are used. If you need the DOMRect sizes call `.getBoundingClientRect()` on the
`dom` prop from [useThrelte](/docs/reference/core/use-threlte#usage)

You can wrap your `<Canvas>` in an HTMLElement with `position:fixed` to keep
the size the same.

## Attaching events

Within components you may wish to listen to events, use the `dom` element from
the `useThrelte` hook.
